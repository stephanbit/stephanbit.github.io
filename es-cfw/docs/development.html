# First-time setup

Make sure you have the following globally installed:

  • Node (at least the latest LTS)
  • Yarn (at least 1.0)
  • Vue CLI (at least 3.0)

Then update the following files to suit your application:

  • src/app.config.js (provides metadata about your app)
  • Jenkinsfile (assuming you want to automatically deploy to production with continuous integration)

# Installation

# Install dependencies from package.json
yarn install

# Developer UI

This project use vue cli 3, so you can also use the great Developer UI

# Start developer ui
vue ui

# Dev server

# Launch the dev server
yarn dev

# Launch the dev server and automatically open it in
# your default browser when ready
yarn dev --open

# Launch the dev server with the nightwatch client for
# test-driven development in a friendly interface
yarn dev:e2e

# Developing with the production API

By default, dev and tests filter requests through the mock API in tests/mock-api. To test directly against a local/live API instead, run dev and test commands with the API_BASE_URL environment variable set. For example:

# To develop against a local backend server
API_BASE_URL=http://localhost:3000 yarn dev

# To test and develop against a production server
API_BASE_URL=https://example.io yarn dev:e2e

# Generators (in progress, available in beta release!!!)

This project includes generators to speed up common development tasks. Commands include:

# Generate a new fun component with adjacent unit test
eze create component

# Generate a new view with adjacent unit test
eze create view

# Generate a new layout with adjacent unit test
eze create layout

# Generate a new Vuex module with adjacent unit test
eze create module

# Generate a new utility function with adjacent unit test
eze create util

Update existing or create new generators in the generators folder, with help from the eze cli docs.

# Aliases

To simplify referencing local modules and refactoring, you can set aliases to be shared between dev and unit tests in aliases.config.js. As a convention, this project uses an @ prefix to denote aliases.

# UI Components (in progress, available in beta release!!!)

UI Components = Base Components. To keep the the ui layer highly decoupled and configurable, we use vue cli plugins. That makes it possible to easly switch the ui layer of the client framework for propper platform or devices.

# UI Components Development

One of the key feature for the client framework is reusability. To develop and test components in isolation (outside of the app) we are using Storybook. More information about configuration or how to create your own set of ui components in one of our next releases.
